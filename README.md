# Методика оцінки ризиків та загроз інформаційної безпеки РАЗОМ

Назва методики сформована як абревіатура від «**Р**изики», «**А**ктиви», «**З**агрози» та «**О**ціночна **М**одель» – основних її компонентів, що задіяні в оцінки сценаріїв кіберзагроз. Назва методики також підкреслює важливість повноцінного долучення всіх суб’єктів до процесу ідентифікації та оцінювання ризиків відповідної інформаційно-комунікаційної системи.

# Мета
Головною метою створення методики є надання установам, що задіяні у створені, модернізації та масштабуванні інформаційно-комунікаційних систем, зрозумілого та дієвого механізму з проведення оцінки ризиків як частини вправи щодо створення цільового профілю безпеки для інформаційно-комунікаційної системи, такого що відповідає сучасним вимогам кібербезпеки.
В межах запропонованої методики надана базова таксономія (категоризація та приклади компонентів та атрибутів що використовуються в оцінюванні). Важливо розуміти, що як запропонована методика, так і відповідна таксономія не є обов’язковими для використання, а для проведення оцінки ризиків може використовуватись:
* запропонована методика РАЗОМ чи її адаптація під потреби конкретної установи;
* будь-яка існуюча міжнародно визнана методологія (NIST SP 800-30, IEC 31010, ISO/IEC 27005, тощо) чи існуючі україномовні стандарти, що на них базуються (наприклад, ДСТУ ISO/IEC 27005);
* власна методика з оцінки ризиків інформаційної безпеки (за умови забезпечення належного рівня управління ризиками).

Відповідне рішення залишається за керівником відповідної установи.

# Формат розповсюдження
Методика розповсюджується у вигляді чотирьох файлів:
* описової частини,
* табличного файлу (розрахункової моделі, що використовується для первинного налаштування атрибутів моделі, наповнення даними та фіксування результатів оцінки) (для коректної роботи файлу потрібен табличний редактор з підтримкою формул динамічних масивів (наприклад, MS Excel версії не нижче 2019 чи його безкоштовні аналоги),
* табличного файлу з передзаповненими фіктивними даними (лише для прикладу),
* шаблону звіту з оцінки ризиків.
